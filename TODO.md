# 代办事项

## 项目描述

该项目的目的为支持 windows、macOS、linux 等三个系统的共享剪切板桌面应用。需要支持纯文本、HTML、RTF、图片、文件等。
关于文本、较小尺寸的图片等可以通过http、https协议进行共享。大尺寸图片、文件等则使用 WebDAV 协议进行文件共享和获取。可以考虑使用 dufs 库.


## 代办项
[ ] config.toml 文件，该文件用于记录用户对应用的设置，例如快捷键、窗口默认位置、窗口大小等。

[ ] 前端使用 React + vite + shadcn/ui + Radix Primitives，根据后续项目情况考虑是否使用状态管理。

[ ] 目前窗口应该有：剪切板、设置页、文件共享窗口。

[ ] 【Windows、Mac OS、Linux】用户在通过快捷键唤出剪切板窗口时，获取当前所聚焦的窗口的位置，判断是否在文本输入框中，如果聚焦在文本输入框，则将剪切板窗口显示光标下方、否则显示在一个默认位置。
    Windows 可以通过将输入法框架（IMM）窗口注册为输入法候选窗、例如输入法机制“寄生”焦点、或低阶键盘钩子等方式实现类似的方式。Mac OS 和 Linux 待补充。

[ ] sqlite：用于记录和管理用户需要共享的剪切板内容，sqlite 用于记录存储共享的文本、图片、文件地址等。

[ ] dufs：用于文件共享系统。

[ ] clipboard-rs库：暂时考虑使用该库进行监听剪切板的更新，如果对于复制、粘贴等操作与系统存在冲突时需要自己定制修改该库（例如点击剪切板时，需要进行粘贴操作，会将点击粘贴的文本先写入到剪切板中，
    这样会导致触发 clipboard-rs 的监听事件，此时就需要对 clipboard-rs 进行一个定制，避免重复触发监听事件）
